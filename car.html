Car Price Prediction

A Flask-based Machine Learning web application designed to accurately predict the resale price of used cars based on a multitude of defining characteristics. This innovative web app caters to both sellers and buyers, providing them with a reliable estimate of a car's value prior to any transaction. Hosted on the Heroku Cloud Platform, this user-friendly application requires input information such as the purchase year, ex-showroom price, mileage, and more.

Project Overview:
Commencing this project, I initiated by conducting a comprehensive analysis of the dataset. This involved determining the dataset's dimensions, identifying unique values within specific columns, and addressing missing data points. Notably, I engineered a new feature named "No of Years" by computing the difference between the current year and the year of purchase. Consequently, the original "Year" column was deemed superfluous and was subsequently dropped.
To handle categorical variables, I employed the pandas library to transform these variables into numerical vectors using the get_dummies() function. To ensure optimal model performance, I partitioned the dataset into training and testing subsets, adopting an 80:20 ratio. Given the regression nature of the problem, I employed the RandomForestRegressor model for training. To fine-tune hyperparameters and identify the most optimal model, I harnessed the power of RandomizedSearchCV.

Model Evaluation:
To quantify the efficacy of the model, I employed several key metrics including Mean Absolute Error, Mean Squared Error, and Root Mean Squared Error. These metrics enabled a comprehensive assessment of the model's predictive accuracy and generalization capabilities.

Serialization and Deployment:
To seamlessly integrate the trained model into the Flask web app, I serialized it using the pickle library. This streamlined the model's utilization within the application, enabling efficient and accurate predictions. Users input new data values via the application's form, which are subsequently utilized by the model to generate predictions. These predictions are then presented to the user in an easily comprehensible format.

Learning and Challenges:
This project presented an excellent opportunity for me to delve into the realm of Flask-based development. While it initially posed challenges, my commitment to learning proved invaluable. I diligently studied tutorials and conducted thorough research, enabling me to successfully build this web app. The experience bolstered my proficiency in web development and machine learning integration.

Room for Improvement:
One notable aspect of improvement lies in the calculation of the number of years. Although the current implementation uses hard-coded values, a more elegant solution could involve leveraging datetime functions or employing configuration files such as YAML. This modification would enhance the application's flexibility and future-proof its adaptability.
